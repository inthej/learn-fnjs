# go - go는 특정 리스트를 축약해 나가는 코드를 만들적에 사용합니다. 축약하기 때문에 go 함수는 결국 reduce 입니다. reduce 인데 인자로 전달받은 함수들의 실행 결과를 축약해서 리스트에 전달된 함수들을 수행할 수 있습니다. go 함수를 통해서 중첩된 함수사용에 대한 코드를 거꾸로 읽는게 아니라..... 정방향으로 읽을 수 있는 이점을 가지게 됩니다. 이로서 의식의 흐름대로 코드를 작성할 수 있어 가독성을 올릴 수 있습니다. # 실수 리듀서 함수안에서 합산방식을 결정하는 실수를 하였다. acc += a 어떻게 축약해 나갈지 자체도 함수를 사용하는 외부세상에서 할 수 있도록 처리해야 한다. acc = f(acc, a) 나도 모르게 작성한 코드였으며... 실수를 통해서 함수형 사고가 부족하다고 느꼇다.. 함수형 사고방식을 좀 더 갖을 필요성을 느꼇다 모든.. 함수를 사용하는 외부세상에서 결정 할 수 있도록 추상화해야 한다.
